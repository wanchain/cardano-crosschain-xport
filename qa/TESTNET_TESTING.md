## What works today

### Outbound: EVM → Cardano (partial)

**What works:** Calling `tokenHome.send()` on Wanchain Testnet dispatches a `MessageDispatchedNonEvm` event on the gateway. Tokens are locked in TokenHome.

**What doesn't work yet:** The message is not delivered to Cardano because there are no active Storeman Groups relaying Cardano-bound messages on testnet. See "Enabling full E2E" below.

**Verified tx:** `0x491e9b66f9d5dee4f46217062aa16fe5b79048eb9f93d4a2598d7f2e772e6d54`

### Inbound: Cardano → EVM (not tested on testnet)

Requires Storeman Groups to call `gateway.receiveMessageNonEvm()` with a valid MPC signature. Cannot be tested without Wanchain team involvement.

## How to test outbound (EVM → Cardano dispatch)

### Prerequisites

- Funded Wanchain Testnet wallet (get WAN from https://wanchain-faucet.vercel.app)
- Node.js >= 18

### Steps

```bash
cd evm-contracts
yarn install

# Set your Wanchain Testnet private key
export PK=<your-wanchain-testnet-private-key>

# Verify you have XTokens (our wallet has 100M)
node -e "
const { ethers } = require('ethers');
const p = new ethers.providers.JsonRpcProvider('https://gwan-ssl.wandevs.org:46891');
const abi = ['function balanceOf(address) view returns (uint256)'];
const token = new ethers.Contract('0x5745e9BcCF4843beD25f871B4Bbe1f2AE54e2B73', abi, p);
token.balanceOf('<your-address>').then(b => console.log('Balance:', b.toString()));
"

# If you need XTokens and don't have any, deploy your own:
npx hardhat --network wanchainTestnet run scripts/deploy_XToken_forTest.js
# Then redeploy TokenHome with the new XToken address (update wanTestnet.json first)
```

### Send tokens to Cardano

```bash
node -e "
const { ethers } = require('ethers');
const provider = new ethers.providers.JsonRpcProvider('https://gwan-ssl.wandevs.org:46891');
const wallet = new ethers.Wallet(process.env.PK, provider);

const xTokenAddr = '0x5745e9BcCF4843beD25f871B4Bbe1f2AE54e2B73';
const tokenHomeAddr = '0x5b7E38C4c317D8054e328A1DE679c3Fd91E758c0';

async function send() {
    // Approve
    const erc20 = new ethers.Contract(xTokenAddr, ['function approve(address,uint256) returns (bool)'], wallet);
    await (await erc20.approve(tokenHomeAddr, ethers.constants.MaxUint256)).wait();

    // Build CBOR for Cardano receiver
    const PlutusUtil = require('./utils/plutusDataTool.js');
    const pu = new PlutusUtil();
    const cardanoReceiver = '<valid-cardano-preprod-address>';
    const plutusData = pu.genBeneficiaryData(cardanoReceiver, 10000);

    // Send
    const tokenHome = new ethers.Contract(tokenHomeAddr, ['function send(bytes) external'], wallet);
    const tx = await tokenHome.send(plutusData, { gasLimit: 800000 });
    const receipt = await tx.wait();
    console.log('TX:', tx.hash, 'Status:', receipt.status === 1 ? 'SUCCESS' : 'FAILED');
}
send();
"
```

**Important:** The Cardano receiver address must have a valid bech32 checksum. Use an address generated by `msg-agent/scripts/setup-preprod.ts` or a Cardano wallet.

## How to test Cardano operations on Preprod

### Prerequisites

- Blockfrost Preprod API key (https://blockfrost.io)
- `msg-agent/.env` configured (already done — keys and addresses from previous deployment)

### Run the msg-agent monitor

```bash
cd msg-agent
yarn start
```

This watches for inbound tokens at the handler address and outbound tasks at the outbound handler address.

### Create an inbound proof manually (simulates Storeman relay)

```bash
npx ts-node scripts/test-local.ts
```

This uses the Ed25519 test GPK to create a signed inbound proof on Cardano Preprod. The msg-agent monitor will then process it (burn inbound token, mint bridge tokens).

### Create an outbound task (send bridge tokens to EVM)

```bash
npx ts-node -e "ts-node src/index.ts client"
```

Or use the CLI:
```bash
yarn client
```

This sends bridge tokens to the outbound handler with a Beneficiary datum targeting an EVM address.

## Enabling full testnet E2E

To complete the testnet integration, coordinate with the Wanchain team:

### 1. Ask Wanchain to enable Storeman Group relay for Cardano

They need to configure their Storeman Group nodes to:
- Watch `MessageDispatchedNonEvm` events on the gateway with `toChainId = 2147485463`
- Relay signed proofs to our Cardano `InboundMintCheck` address
- Watch our Cardano `xport` address for outbound messages
- Call `gateway.receiveMessageNonEvm()` with MPC signatures for inbound Cardano→EVM messages

### 2. Update GPK to Storeman Group's public key

Once Wanchain provides their Storeman Group's public key:

```bash
cd msg-agent
# Set the Storeman Group's public key in .env or pass via INBOUND_SIGNING_KEY
INBOUND_SIGNING_KEY=<storeman-group-public-key-hex> npx ts-node scripts/update-gpk.ts
```

This is a single GroupInfoParams datum update — no redeployment needed.

### 3. Mint more check tokens (if needed)

Each inbound proof consumes one check token. To mint more:

```bash
npx ts-node scripts/mint-check-tokens.ts
```

Requires AdminNFT in the deployer wallet.

## Architecture reminder

```
EVM (Wanchain Testnet)                    Cardano (Preprod)
──────────────────────                    ─────────────────
User → TokenHome.send()                  InboundMintCheck ← check token
         ↓                                      ↓
WmbGateway.dispatchMessageNonEvm()       InboundToken minted at handler
         ↓                                      ↓
MessageDispatchedNonEvm event            msg-agent: burn inbound, mint bridge tokens
         ↓                                      ↓
   [Storeman Group relay]               Bridge tokens at receiver
         ↓                              ─────────────────
   Signed proof on Cardano              User sends bridge tokens to outbound handler
                                                 ↓
                                        msg-agent: burn bridge, mint outbound at xport
                                                 ↓
                                        [Storeman Group relay]
                                                 ↓
                                        gateway.receiveMessageNonEvm()
                                                 ↓
                                        TokenHome releases XTokens to EVM receiver
```
